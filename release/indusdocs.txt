API Reference
Use the BASE URL with the endpoints below to integrate and interact with Indus Appstore APIs (eg: https://developer-api.indusappstore.com/devtools/apk/upgrade/com.example.myapp).
Each endpoint comes with a detailed request structure, parameters and response format to help you get started quickly.
Note: Old BASE URL (https://developer-api.indusappstore.com/apis/indus-developerdashboard-service) will be deprecated soon. Please use the new BASE URL for all your API requests.
BASE URL
https://developer-api.indusappstore.com

AAB Upgrade/Upload
This API helps you to update your Android App Bundle (AAB) file and Java Keystore (JKS) file.
This API requires the AAB file, keystore file, and associated credentials as mandatory fields to sign and validate the app bundle.
Using the AAB format allows for more efficient app distribution and installation, as it enables dynamic delivery of features and resources based on the users device configuration.
Dont worry we dont store your keystore values, we only use it to sign the AAB file.
Parameter	Description
package
The unique package name of the app you want to upgrade. (eg: com.example.myapp)
file
An array of binary files containing AAB file & JKS (keystore) file.
keyPassword
The password for the private key in the keystore.
keystoreAlias
The alias name of the keystore that has been configured.
keystorePassword
The password required to access the protected keystore file.
message

Errors
We use standard HTTP status codes to indicate the success or failure of an API request.
The API will return a JSON response with an error message and a status code indicating the type of error.
Every error response will include a message field that provides more details about the error.
These errors are common for all endpoints
Parameter	Description
2xx
The request was successful.
4xx
Your request has failed due to error in data provided. This could be due to invalid input, authentication failure, or other issues.
5xx
Your request has failed due to a server error. This is rare and usually indicates a problem with the API itself. Please try again later.
ERROR CODES
200: This is successful request.
400: The data provided in request is invalid. Please check you request body.
403: Authentication failed. Please check your API key and try again.
404: The app you are trying to access does not exist. please recheck the package name.
409: The app is already in review. Please wait for the review to complete before making any changes.
433: The File uploaded doesnt match the required format. Please check your file type and try again.

Cancel App Review
The Cancel App In-Review API allows you to cancel an app review that is currently in progress. This is useful when you need to make changes to the app or its metadata before it is reviewed or published.
This API ensures that you can halt the review process when necessary, giving you the flexibility to make updates or corrections before resubmitting the app for review.
You can find the id of the version you want to cancel in the response of the Get All App Versions API.
Parameter	Description
package
The unique package name of the app you want to upgrade. (eg: com.example.myapp)
